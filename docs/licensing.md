# Лицензионный аудит RemoteDesk

## Основная лицензия проекта

**AGPL-3.0** — унаследована от RustDesk.

⚠️ **Ключевое ограничение AGPL**: если приложение используется для сетевого взаимодействия
(что является основной функцией RemoteDesk), исходный код должен быть доступен пользователям.
Для SaaS/коммерческого использования требуется юридический аудит.

## Стратегия митигации рисков

### AGPL (RustDesk)
1. **Текущий подход**: проект открыт под AGPL-3.0, весь код доступен
2. **При коммерциализации**: 
   - Рассмотреть замену AGPL-компонентов на альтернативы с permissive лицензиями
   - Использовать RustDesk как отдельный процесс («aggregation», не «derivative work»)
   - Проконсультироваться с юристом по software licensing

### GPL (кодеки)
1. **x264/x265**: исключены из проекта, используем OpenH264 (BSD) + аппаратные кодеки
2. **FFmpeg**: использовать только через динамическую линковку (LGPL compliance)
3. **Аппаратные кодеки** (NVENC, AMF, VideoToolbox, VAAPI): не требуют лицензионных отчислений

### libjuice
- Лицензия требует проверки перед интеграцией (Фаза 2)
- Альтернатива: `webrtc-rs` (MIT/Apache-2.0) или прямая работа с STUN/TURN

## Проверка автоматическая

В CI/CD используется `cargo-deny` (см. `deny.toml`):
- Запрещены: GPL-3.0 (только для списочных пакетов), неизвестные лицензии
- Предупреждения: LGPL-3.0, CDDL-1.0
- Разрешены: MIT, Apache-2.0, BSD, ISC, MPL-2.0, AGPL-3.0

## Рекомендация

Для коммерческого closed-source продукта необходимо:
1. Заменить все AGPL-компоненты (переписать с нуля или найти MIT/Apache альтернативы)
2. Отказаться от GPL/LGPL статической линковки
3. Получить заключение юриста по software licensing

**На текущем этапе (MVP) проект полностью Open Source под AGPL-3.0.**
