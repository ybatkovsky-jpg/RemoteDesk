# Архитектура RemoteDesk

## Обзор

RemoteDesk — кроссплатформенное приложение удаленного доступа, построенное на базе
ядра **RustDesk** (AGPL-3.0) с UI-слоем на **Tauri v2** (MIT/Apache-2.0).

## Принципиальная схема

│  │   - Remote view      │◄──►│  - session control           │   │
│  │   - Peer list        │    │  - display enumeration       │   │
│  │   - Settings         │    │  - input forwarding           │   │
│  └──────────────────────┘    └──────────┬───────────────────┘   │
│                                         │                        │
├─────────────────────────────────────────┼────────────────────────┤
│                     Rust Core (AGPL)    │                        │
│  ┌──────────────────────────────────────▼────────────────────┐  │
│  │  src-tauri/src/commands/  ← Tauri → Core bridge           │  │
│  └───────────────────────────────────────────────────────────┘  │
│                                                                  │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────────────┐   │

## Слои

### 1. Frontend (SolidJS + TypeScript)
- Отвечает за UI: список пиров, окно удаленного доступа, настройки
- Взаимодействует с Rust-бэкендом через Tauri `invoke()` и `listen()` (events)
- Видео-поток рендерится через Canvas/WebGL (в Фазе 1)

### 2. Tauri Bridge (src-tauri/src/commands/)
- Тонкий слой, преобразующий Tauri команды в вызовы внутренних крейтов
- Управляет состоянием приложения (`AppState`)
- Транслирует события из сетевого слоя во фронтенд через `app_handle.emit()`

### 3. Rust Core (crates/)
- **screen-capture**: Захват экрана (DXGI / ScreenCaptureKit / PipeWire)
- **codec**: Кодирование/декодирование видео (H.264/H.265/VP9)
- **input-sim**: Инжекция клавиатуры/мыши/буфера обмена
- **network**: P2P-соединения, NAT traversal, rendezvous протокол
- **hbb-common**: Общие типы, protobuf-сообщения, конфигурация

### 4. Серверная инфраструктура
- **hbbs**: Rendezvous/signaling сервер (из RustDesk)
- **hbbr**: Relay сервер (из RustDesk)
- **Coturn**: TURN сервер для fallback-релея

## Поток данных

### Стриминг экрана (Host → Control)
```
Screen Capture → Raw Frame (BGRA) → Encoder (H.264/H.265) →
Encrypted P2P Channel → Decoder → Canvas/WebGL Render
```

### Ввод (Control → Host)
```
DOM Events → Tauri invoke → KeyEvent/MouseEvent →
Encrypted P2P Channel → OS Input Injection
```

### Передача файлов
```
File Select → Chunked Read → tus Protocol →
Encrypted Channel → Chunked Write → SHA-256 Verify
```

## Ключевые архитектурные решения

| Решение | Причина |
|---------|---------|
| **Tauri вместо Flutter** | Прямая интеграция с Rust-крейтами без FFI-моста |
| **SolidJS вместо React** | Лучшая производительность при частых обновлениях (видео-поток) |
| **Форк RustDesk, не с нуля** | Готовое ядро захвата/кодирования/сети, проверенное в продакшене |
| **Custom protocol `stream://` для видео** | Обход IPC-бутылочного горлышка для кадров |
| **OpenH264 вместо x264** | BSD-лицензия безопаснее GPL для коммерческого использования |
| **Монорепо с workspace** | Единая сборка, общий Cargo.lock, проще рефакторинг |
