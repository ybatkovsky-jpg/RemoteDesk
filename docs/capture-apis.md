# Screen Capture APIs — исследование

## Сравнение по платформам

### Windows

| API | Латентность | CPU | Оконный захват | Сложность |
|-----|------------|-----|----------------|-----------|
| **DXGI Desktop Duplication** | 10-30ms | <5% | ❌ (только монитор) | Высокая |
| **Windows.Graphics.Capture** | ~30ms | ~3% | ✅ | Средняя |
| **GDI (BitBlt)** | 50-100ms | ~70% | ✅ | Низкая |

**Выбор RustDesk**: DXGI Desktop Duplication — минимальная латентность.
**Альтернатива**: scap использует Windows.Graphics.Capture для простоты.

### macOS

| API | Латентность | CPU | Аудио | Минимальная версия |
|-----|------------|-----|-------|-------------------|
| **ScreenCaptureKit** | 15-30ms | <5% | ✅ | macOS 12.3+ |
| **CGDisplayCreateImage** | 30-60ms | ~30% | ❌ | macOS 10.x+ |

**Выбор**: ScreenCaptureKit — современный, аппаратно-ускоренный, рекомендован Apple.

### Linux

| API | Display Server | Производительность | Безопасность |
|-----|---------------|-------------------|--------------|
| **PipeWire** | Wayland | Хорошая | Медиировано композитором |
| **MIT-SHM** | X11 | Отличная | Неограниченный доступ |
| **XDG Desktop Portal** | Оба | Хорошая | Согласие пользователя |

**Выбор**: MIT-SHM для X11, PipeWire + Portal для Wayland.

## Особенности по платформам

### Windows
- Требуются права администратора для SendInput на некоторых системах
- DXGI Desktop Duplication не работает с защищённым контентом (DRM)
- Для захвата окна логина (UAC) нужен отдельный процесс-сервис

### macOS
- Обязательный запрос разрешения Screen Recording (TCC)
- Начиная с macOS 14, запрос повторяется периодически
- Нет доступа к окнам с защищённым контентом (DRM)

### Linux
- X11: неограниченный доступ к любому окну (проблема безопасности)
- Wayland: PipeWire требует xdg-desktop-portal и согласия пользователя
- Для headless-систем нужен виртуальный дисплей
