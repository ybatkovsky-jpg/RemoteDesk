## Выбор для RemoteDesk

### Приоритет выбора кодека
1. **H.264 HW** (NVENC/AMF/QSV/VideoToolbox) — если доступен
2. **H.265 HW** — если доступен и поддерживается обоими клиентами
3. **OpenH264 (software)** — fallback, BSD-лицензия
4. **VP9 (software)** — последний fallback

### Обоснование
- **OpenH264 (BSD)** предпочтительнее x264 (GPL) для снижения лицензионных рисков
- **Аппаратное кодирование** даёт минимальную задержку и нагрузку на CPU
- **VRAM path** от RustDesk (Windows): zero-copy из DXGI в NVENC/AMF/QSV
- **VP9** — программный fallback, зрелый, BSD-лицензия

## Настройки для реального времени

| Параметр | Значение | Примечание |
|----------|----------|------------|
| B-frames | 0 (выкл) | Минимизация задержки |
| Delay | 0 | Немедленная отправка |
| Rate Control | CBR/VBR | Постоянный/переменный битрейт |
| GOP size | 60-120 | Ключевой кадр каждые 2-4 сек |
| Preset | fast/medium | Баланс качество/скорость |
| Битрейт | 2-10 Mbps | Адаптивный, зависит от разрешения |
